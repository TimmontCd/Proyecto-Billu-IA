Diagramas de Arquitectura y Flujos de Billú

Proyecto: Billú / Control Tower Directiva

Fecha de corte: 15 de abril de 2026

Base analizada: repositorio Apps Script, módulo Node/TypeScript de WhatsApp, documentación técnica institucional y manifiestos de operación en el workspace del proyecto.

Propósito: documentar arquitectura, flujos operativos, componentes, dependencias y puntos de integración para facilitar institucionalización, soporte, continuidad y transferencia operativa.

1. Resumen ejecutivo

La plataforma Billú está construida principalmente como una Web App en Google Apps Script con almacenamiento operativo sobre Google Sheets, integración documental con Google Drive / Google Docs y conectores hacia APIs externas como Google Analytics, correo, OpenAI y Meta/WhatsApp. El proyecto también incorpora un backend evolutivo en Node.js + TypeScript + Express para institucionalizar el agente de WhatsApp en una arquitectura más robusta.

Desde el punto de vista funcional, la solución ya cubre monitoreo en tiempo real, resúmenes ejecutivos de cuentas, análisis de uso y cohortes, correlación de depósitos y cargos, categorización de clientes, PMO, task tracking, campañas y asistentes operativos. La base sigue una arquitectura por capas con controllers, services, repositories, views y un core de autenticación, configuración, bootstrap y triggers.

8. Riesgos y observaciones de diseño

  • La solución gana velocidad por vivir sobre Google Sheets, pero hereda límites de concurrencia, consistencia y gobierno de cambios estructurales.
  • La capa de servicios está bien separada, lo cual facilita migraciones graduales hacia backend dedicado o base de datos institucional.
  • El uso de BaseRepository como acceso genérico ayuda a estandarizar operaciones, pero centraliza mucho comportamiento crítico en una sola abstracción.
  • El agente de WhatsApp ya tiene dos arquitecturas coexistiendo: Apps Script como PoC funcional rápida y Node como arquitectura objetivo más segura.
  • Los jobs críticos siguen dependiendo de triggers de Apps Script; institucionalmente conviene migrar los más sensibles a un scheduler administrado y auditable.
  • La relación entre dashboards ejecutivos, campañas y motores de tarea ya forma una base real de torre de control operativa, no solo un tablero visual.
