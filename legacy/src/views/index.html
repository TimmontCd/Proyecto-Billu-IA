Lógica outbound

1Lee `CampañaEnrolamiento` y detecta elegibles por teléfono, opt-in y estado.
2Envía template o texto y guarda trazabilidad de envío, error y message id.
3Si no hay nuevos elegibles, permite reintento sobre enviados según regla actual.

Cómo lo vamos a usar

1Si viene de Meta, el bot lo recibe y lo orienta desde el primer mensaje.
2Si ya está en proceso de apertura, el mensaje usa su `Siguiente Paso` para ayudarlo a avanzar.
3Si el cliente pide ayuda humana o el caso se complica, entra a la cola de escalamiento.

Mapa de calor para prototipos nuevos

Estamos corriendo un primer piloto controlado sobre un flujo móvil de 3 pantallas del enrolamiento N2: Aviso de privacidad, Datos de contacto y Código correo. El objetivo es medir zonas tocadas, coordenadas, intentos bloqueados y tasa de éxito antes de llevar esta lógica al flujo completo.

Definición de éxito
Completar las 3 pantallas y confirmar el código

Esta vista queda enfocada solo en la prueba desde móvil. Inicia la sesión, revisa el mensaje de arranque, recorre las 3 pantallas y al terminar verás un mensaje de cierre antes de pasar a resultados.

Qué puede hacer BeBot

Esta primera versión está pensada para ayudarte a operar el tablero de actividades sin cambiar de pantalla. La configuración técnica queda resuelta por detrás para que el usuario solo vea la experiencia útil.

Crear actividades
Ejemplo: "Crea una actividad para Andrea de validar saldos mañana".
Actualizar estatus
Ejemplo: "Marca como completada la actividad seguimiento semanal".
Consultar pendientes
Ejemplo: "Qué pendientes tengo hoy" o "Qué tareas están vencidas".
Resumir el tablero
Ejemplo: "Dame el resumen del equipo" o "Qué sigue esta semana".
